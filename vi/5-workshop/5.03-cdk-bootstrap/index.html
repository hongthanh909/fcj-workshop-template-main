CDK Bootstrap

Tổng quan

AWS CDK Bootstrap chuẩn bị tài khoản AWS của bạn để triển khai các ứng dụng CDK bằng cách tạo các tài nguyên hạ tầng cần thiết. Đây là quy trình thiết lập một lần cho mỗi tổ hợp tài khoản AWS/region, tạo nền tảng cho tất cả các triển khai CDK.

CDK Bootstrap tạo những gì

Quy trình bootstrap cung cấp các tài nguyên AWS sau trong tài khoản của bạn:

1. S3 Bucket (CDK Assets Bucket)

  • Lưu trữ các gói triển khai Lambda function
  • Lưu trữ các CloudFormation templates
  • Lưu trữ các file assets được tham chiếu bởi CDK stacks
  • Mẫu đặt tên: cdk-hnb659fds-assets-{ACCOUNT-ID}-{REGION}
  • Versioning được bật để hỗ trợ rollback

2. CloudFormation Stack (CDKToolkit)

  • Stack hạ tầng chính chứa tất cả tài nguyên bootstrap
  • Tên stack: CDKToolkit
  • Quản lý vòng đời của các tài nguyên bootstrap

3. IAM Roles

  • cdk-hnb659fds-cfn-exec-role: Role thực thi CloudFormation để triển khai stacks
  • cdk-hnb659fds-deploy-role: Role được CDK CLI sử dụng trong quá trình triển khai
  • cdk-hnb659fds-file-publishing-role: Role để upload assets lên S3
  • cdk-hnb659fds-lookup-role: Role để đọc environment context

4. SSM Parameters

  • /cdk-bootstrap/hnb659fds/version: Lưu trữ số phiên bản bootstrap
  • Được sử dụng để kiểm tra tương thích

Tổng quan Infrastructure Stacks

Sau khi bootstrap, dự án của chúng ta sẽ triển khai nhiều CDK stacks được định nghĩa trong D:\Project_AWS\everyonecook\infrastructure\lib\stacks:

  • auth-stack.ts: Hạ tầng xác thực và phân quyền (Cognito)
  • backend-stack.ts: Các dịch vụ backend core và Lambda functions
  • certificate-stack.ts: Quản lý chứng chỉ SSL/TLS
  • core-stack.ts: Tài nguyên hạ tầng core
  • dns-stack.ts: Cấu hình Route 53 DNS
  • observability-stack.ts: Giám sát, logging, và observability

Các stacks này phụ thuộc vào hạ tầng bootstrap để triển khai thành công.

Hiểu về Bootstrap Resources

Chi tiết S3 Bucket

CDK assets bucket đóng vai trò là kho lưu trữ trung tâm cho tất cả các artifacts triển khai:

  • Mục đích: Lưu trữ CloudFormation templates, Lambda deployment packages, và static assets
  • Vòng đời: Tồn tại qua các lần triển khai, cho phép khả năng rollback
  • Bảo mật: Mã hóa khi lưu trữ, bucket policies giới hạn quyền truy cập cho các roles được ủy quyền
  • Đặt tên: Đặt tên xác định dựa trên account ID và region

1. CloudFormation Execution Role (cfn-exec-role)

  • Được CloudFormation sử dụng để tạo/cập nhật/xóa tài nguyên stack
  • Có quyền rộng để quản lý tài nguyên AWS
  • Trust relationship với CloudFormation service

2. Deploy Role (deploy-role)

  • Được CDK CLI sử dụng trong các thao tác cdk deploy
  • Có thể assume CloudFormation execution role
  • Có quyền upload assets và khởi tạo triển khai

3. File Publishing Role (file-publishing-role)

4. Image Publishing Role (image-publishing-role)

5. Lookup Role (lookup-role)

  • Đọc environment context (VPCs, subnets, v.v.)
  • Quyền chỉ đọc cho việc tra cứu tài nguyên
  • Được sử dụng trong quá trình synthesis cho context queries

Phân tích chi phí

Chi phí Bootstrap Resources

Thiết lập một lần:

  • Tạo CloudFormation stack: Miễn phí
  • Tạo IAM roles: Miễn phí
  • SSM parameters: Miễn phí

Chi phí hàng tháng:

Tài nguyênSử dụngChi phí (Ước tính)
S3 Storage< 1 GB (thông thường)$0.023/GB = ~$0.02/tháng
S3 RequestsTối thiểu (GET/PUT)~$0.01/tháng
IAM RolesKhông tính phí$0.00

Tổng chi phí ước tính: ~$0.03/tháng (không đáng kể)

Lưu ý: Đối với các ứng dụng serverless, chi phí vẫn tối thiểu vì chúng ta chỉ lưu trữ Lambda deployment packages trong S3.
